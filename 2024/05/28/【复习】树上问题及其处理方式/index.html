Link Cut Tree / Euler Tour Tree

LCT

回忆树链剖分里面我们干的事情，我们把树分解成若干条链，方便维护信息。

但这种剖分的结构是在整棵树一开始的时候就已经确定的，没办法在树的结构变化时动态维护，为维护带来了诸多麻烦。

既然树链没法固定，让它随着询问变化如何？

先从简单的询问开始考虑，既然是链剖分，那么先考虑树链的询问，假设所有询问都是 uuu 到根的，并且我们已经得到了这样的一个链剖分结构，每条链都是一段从上到下的链（即链的 LCA 是其一个端点），并在每条树链都有某个结构来维护信息：

我们要查询红色一段的信息，在树链剖分中，我们对这东西的处理方法就是把这几条树链的信息作合并，由于树链的条数是 O(log⁡n)\mathcal O(\log n)O(logn) 条的，询问复杂度总是有着保障。

但是我们这种新的链剖分结构相当难以保障这种性质，即使想要保证向上常数步，子树大小翻倍等性质也是困难的。

那么用均摊如何？仿照 Splay 每次询问一个点就旋转到根，那么我们每次询问一个 uuu 的时候把询问的链并成一条，然后裁掉每条链不在 uuu 到根的路径上的部分。从 uuu 开始，每次裁掉末尾，把它和“当前链”接在一起。

我们把形如这样的操作成为 Access⁡(u)\operatorname{Access}(u)Access(u)。

我们一开始说过，每条链都是从上到下的，那么我们的结构就要：

  • 维护的是所有点按深度排序形成的序列
  • 支持分裂为一段前缀和一段后缀
  • 支持合并两段值域不交的序列

看起来权值线段树与平衡树都符合我们的条件。

那么维护 u,vu,vu,v 间链的信息呢？

一个暴力的想法是，既然我们会了查询 uuu 到根的信息，那我们干脆就把 vvv 转到根。大多数的路径信息是可以把树看作无根树的，随便转就行了。

咋做呢。

我们先 Access⁡(v)\operatorname{Access}(v)Access(v)。现在这个树应该长左边这样：

我们要让 vvv 变到根，那么就是让 vvv 在链上的深度从最大变成最小。

直接对 vvv 现在在的这条链作区间翻转。

并且我们惊喜地发现，黑链（子树内）与蓝链（子树外）不用做任何修改。

所以我们现在知道了，维护信息的结构要：

  • 维护的是所有点按深度排序形成的序列
  • 支持分裂为一段前缀和一段后缀
  • 支持合并两段值域不交的序列
  • 支持区间翻转

显然是平衡树。

并且，我们应当使用 Splay 而非其他平衡树，因为 Splay 在均摊中有着良好的性质（天然支持 Finger Search）。

可以证明，点数为 nnn，操作数为 mmm 时，复杂度是均摊 O(nlog⁡n+mlog⁡n)\mathcal O(n\log n+m\log n)O(nlogn+mlogn) 的。

简要实现，以及更多操作

上面已经说了，我们 Access⁡\operatorname{Access}Access 的过程要每次切断当前链的末端，找到当前链链顶父亲所在的链，合并两条链。

一、三操作用 Splay 实现是简单的，但是二操作还需要我们对每条链维护原树上链顶节点的父亲节点。我们将其记在这条链的 Splay 的根节点，形成一个链间认父不认子的 Splay 森林。

偷两张 OI wiki 的图

这是原树上的链剖分结构。

这是 Splay 森林的结构。

以 CGHJ\text{CGHJ}CGHJ 这一条链为例，图二中的实边是这一条链上的 Splay 的结构，其根节点 G\text{G}G 的父亲指针指向了链顶 C\text{C}C 的父亲 A\text AA，但节点 A\text{A}A 不会将 G\text{G}G 作为其任何一个儿子。这就是认父不认子的含义。这个 Splay 森林也被称为 AuxTree（辅助树）。

有了这样的结构后，实现动态加边是平凡的，只要将点直接挂上去，均摊就会为我们解决剩下的一切。

动态删边也很简单，然后将 uuu 或者 vvv 转到 Splay 的根，如果有边，那么一个就是另一个的右儿子，并且其左儿子是没有节点的。直接把儿子和父亲删掉就行了。

还可以判断两点是否连通等等，具体实现看模板题的代码。加了一些注释。

Top Cluster 与 Top Tree

我们想要在树上同样拥有像序列线段树那样的二叉分治结构以便合并信息，又不希望像边分治那样引入多余的边与信息给维护带来麻烦。
