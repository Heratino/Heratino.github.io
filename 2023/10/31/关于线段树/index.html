线段树维护分治信息

通过分治维护区间半群修改查询的视角，我们可以更为清晰审视线段树所维护信息的构造过程，在一定程度上规避人类智慧防止我想不出来。

什么是范围修改查询？

根据 lxl 的定义，范围修改查询是这样的一个问题：

给一个初始集合 I0I_0I0​，一个有限集合 I⊂I0I\subset I_0I⊂I0​，询问集合 $ Q\subset 2^{I_0}$ 满足 I0∈QI_0\in QI0​∈Q，信息半群 (D,+)(\mathcal D,+)(D,+)，标记半群 (M,∗)(\mathcal M,*)(M,∗)。且二元运算符 ∗:D×M→D*:\mathcal D\times \mathcal M\rightarrow \mathcal D∗:D×M→D.

  • 结合律：∀a∈D\forall a\in \mathcal D∀a∈D，b,c∈Mb,c\in \mathcal Mb,c∈M，(a∗b)∗c=a∗(b∗c)(a*b)*c=a*(b*c)(a∗b)∗c=a∗(b∗c).
  • 分配律：∀a,b∈D\forall a,b\in \mathcal D∀a,b∈D，c∈Mc\in \mathcal Mc∈M，(a+b)∗c=a∗c+b∗c(a+b)*c=a*c+b*c(a+b)∗c=a∗c+b∗c.

初始权值 d0:I→Dd_0:I\rightarrow \mathcal Dd0​:I→D.

这堆东西的意思大概就是信息对信息加法构成一个半群，标记对标记复合，也就是 ∗*∗ 运算构成一个半群，而且标记半群对信息半群具有分配律。每次修改和询问都是修改 I0I_0I0​ 的一个子集。

什么是线段树？

当初始集合可以以某种顺序组织为一个序列，而每次询问的集合在这个序列上总代表一个区间时，由于 D\mathcal DD 在 +++ 下构成一个半群，所以其具有可合并性。

所以一个统计区间信息的做法就是每次过中点把序列分成两半再合并，

Classic Problem(GSS 3)

单点修改，区间最大子段和

区间最大子段和的一个经典 DP 是

fi=max⁡{fi−1+ai,ai}f_i=\max\{f_{i-1}+a_i,a_i\} fi​=max{fi−1​+ai​,ai​}

我们发现这个东西因为是一段前缀，每次修改会影响后面的所有部分。所以考虑分治，每次只会影响到 log⁡\loglog 段区间。我们只要对被影响到的区间重算答案即可。

我们逐个分析重算信息需要哪些量。首先我们必须要维护这个区间的区间最大子段和，为了保证

c(d)=c(d1)+c(d2)c(d)=c(d_1)+c(d_2) c(d)=c(d1​)+c(d2​)

考虑 d1d_1d1​ 和 d2d_2d2​ 需要维护什么信息。首先同样必须维护最大子段和，其次在合并时还要利用到前缀和后缀最大和。

为了维护前缀和后缀最大和，我们还需要利用到什么信息？答案是区间和。而这个信息不需要额外信息就可以自我维护，所以我们只用维护：

  • 区间最大子段和
  • 前缀和后缀最大和
  • 区间和

就可以解决这个问题。

一个 ( 代表向下走一步并新建一个节点，一个 ) 代表向上走一步，每个 ) 会和前面最近的 ( 匹配，一个匹配的括号子串代表一个子树。

那么每个节点没有匹配的一定先是若干个 )，和其后的若干个 (。

形态上就是先向上再向下的链上挂着若干个子树。

考虑所有的区间，每个区间可能的贡献是它的那条先向上再向下的链，直径落在那些子树里的情况下，会被其他的区间统计到。

我们当然可以对每个区间手动把匹配消掉来求出这条链的贡献。但是由于我们最终的目的是把这个过程挂上线段树，这个过程过于复杂了，我们不能这样做。

一个更好的替代是套路地把 ( 看做 +1+1+1，) 看做 −1-1−1 ，设 prei=∑j=liaj,sufi=∑j=irajpre_i=\sum_{j=l}^{i}a_j,suf_i=\sum_{j=i}^{r}a_jprei​=∑j=li​aj​,sufi​=∑j=ir​aj​。则我们有区间 [l,r][l,r][l,r] 的贡献是：

max⁡k=l−1r{sufk+1−prek}\max_{k=l-1}^{r}\{suf_{k+1}-pre_k\} k=l−1maxr​{sufk+1​−prek​}

这给我们的启发是我们可以对一些不可能成为答案的统计以用简单的操作刻画过程。

也就是牺牲一些复杂度降低整个过程的复杂性。

上述的信息实际上是两个相邻区间的差的最大值，也是消去匹配后最长区间的长度。

  • 由右边加上左边的一段，或者左边加上右边的一段拼接而来。考虑上述转移我们得知，在这里，我们需要维护的信息有：
